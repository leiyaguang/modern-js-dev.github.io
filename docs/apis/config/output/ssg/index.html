output.ssg

适用的工程方案
  • MWA
  • 类型： boolean | function
  • 默认值： undefined

开启自控式路由或约定式路由 SSG 功能的配置。

客户端路由

相关内容可以查看自控式路由或约定式路由

示例#

当 ssg 配置设置为 true 时，将会默认开启所有约定式路由的 SSG 功能。

例如 src/ 目录下有以下文件结构：

.├── pages/│   ├── user/│   │   └── info.tsx│   ├── home.tsx│   └── index.tsx├── .eslintrc.json

在 modern.js.config 中做以下设置：

module.exports = {  output: {    ssg: true,  },};

执行 pnpm build 构建应用后。dist/ 目录将会生成三张 HTML 分别对应三条路由（不开启 SSG 时只有一张 HTML 对应主入口），并且可以看到所有 HTML 都已经是渲染之后的。

output.ssg 的配置也可以是钩子函数，每个可以被 Modern.js 识别的路由都会触发一次钩子。

可被识别的路由是指在构建阶段 Modern.js 就能获取到的路由信息，包括每个入口以及约定式路由的每一条客户端路由。

例如以下目录结构：

.├── src/│   ├── contacts/│   │   └── App.tsx│   ├── landing-page/│   │   └── pages/│   │       ├── [user].tsx│   │       ├── docs.tsx│   │       └── index.tsx

钩子函数会被触发 3 次，分别是由：

  1. contacts 入口触发，触发路由为 /contacts
  2. landing-page/index.tsx 触发，触发路由为 /landing-page
  3. landing-page/docs.tsx 触发，触发路由为 /landing-page/docs
  4. landing-page/docs.tsx 触发，触发路由为 /landing-page/:user

默认情况下，所有约定式路由的客户端路由，在设置 output.ssg 配置后都会在构建阶段渲染，可以在函数中返回 false 来取消指定路由的默认行为，例如取消上述 docs 页面在构建时的渲染

module.exports = {  output: {    onSSG(context: PageContext) {      if (context.route.path === '/landing-page/docs') {        return false      }    },  },};

而类似 React Router 的路由则是运行时可知的，无法在构建时被识别，例如：

contacts/App.tsx
import { Switch, Route, Link } from '@modern-js/runtime/router';function App() {  return (    <Switch>      <Route exact={true} path="/home">        <div>Home Page</div>      </Route>      <Route path="*">        <div>404</div>      </Route>    </Switch>  );}export default App;

上述代码中的 /home 路由是运行时的状态，Modern.js 无法自动开启该路由的 SSG。

开发者可以通过 ssg 函数配置，调用 createPage，将该路由添加为新的 SSG 页面，例如：

module.exports = {  output: {    ssg(context: PageContext) {      if (context.route.path === '/contacts/home') {        context.createPage({          output: './home.html',        });      }    },  },};

上述目录结构中还存在 [user].tsx 这样的动态路由，虽然是约定式路由，但无法识别确切的路由，也需要通过 createPage 进行添加：

module.exports = {  output: {    ssg(context: PageContext) {      if (context.route.path === '/landing-page/:user') {        context.createPage({          params: {            user: 'modernjs'          },        });      }    },  },};

PageContext#

type PageContext = {  route: {    path: string,    [propName: string]: string  },  createPage: (pageConfig: PageConfig) => void}

route#

包含触发钩子的路由全部的信息，可以根据该对象进行大部分的逻辑判断。

createPage#

(pageConfig: PageConfig) => void
type PageConfig = {  url?: string,  headers?: Record<string, string>  params?: Record<string, string>  output?: string,}

url#

渲染当前 SSG 页面时，传入的请求路径。

headers#

渲染当前 SSG 页面时，传入的请求头。

params#

渲染当前 SSG 页面时，传入的动态参数。

note

该配置仅在渲染动态路径的约定式路由时使用。

output#

SSG 渲染后页面的文件位置，基于触发钩子的路由所在入口原本渲染的文件位置。

例如上述 contacts/home 路由渲染出的 SSG 页面，指定了 output 为 ./home.html。

原本 contacts 入口的页面默认会渲染到 dist/html/contacts/index.html，因此上述页面将会渲染到 dist/html/contacts/home.html
