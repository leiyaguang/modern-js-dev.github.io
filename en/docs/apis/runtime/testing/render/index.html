render

import { render } from '@modern-js/runtime/testing';

redner 用于渲染组件，并进行测试

函数签名#

function render(  ui: React.ReactElement<any>,  options?: {
  }): RenderResult

简单示例

import { render } from '@modern-js/runtime/testing';
test('renders a message', () => {  const { container, getByText } = render(<Greeting />)  expect(getByText('Hello, world!')).toBeInTheDocument()  expect(container.firstChild).toMatchInlineSnapshot(`    <h1>Hello, World!</h1>  `)})

Options#

container#

类型：DOMElement

默认值：document.body.append(document.createElement('div'))

表示组件所要挂载到的 DOM 节点，默认是会创建一个 div 元素，并自动添加到 document.body 上。这个 div 元素就是组件要挂载的节点。

baseElement#

类型：DOMElement

默认值：如果指定了 container, 则默认值为 container 的值，否则就是 document.body。

用于指定 queries 中使用到的 basename。

hydrate#

类型：boolean

默认值：false

如果设置为 true，则会使用 ReactDOM.hydrate 渲染组件。

wrapper#

类型：React.ComponentType<{children: ReactNode}>

wrapper 是一个 react 组件，可用于自定义渲染逻辑。如

const AllTheProviders = ({ children }) => {  return (    <ThemeProvider theme="light">      <TranslationProvider messages={defaultStrings}>        {children}      </TranslationProvider>    </ThemeProvider>  )}
const customRender = (ui, options) =>  render(ui, { wrapper: AllTheProviders, ...options })

queries#

自定义一些自己的 queries。

const { getByRowColumn, getByText } = render(<MyTable />, {  queries: { ...queries, ...tableQueries },})

RenderResult#

...queries#

所有可用的 queries。

container#

类型：DOMElement

container 是挂载你的 React 组件的 DOM 节点。

baseElement#

类型：DOMElement

debug#

import React from 'react'import { render } from '@modern-js/runtime/testing';
const HelloWorld = () => <h1>Hello World</h1>const { debug } = render(<HelloWorld />)debug()// <div>//   <h1>Hello World</h1>// </div>// you can also pass an element: debug(getByTestId('messages'))// and you can pass all the same arguments to debug as you can// to prettyDOM:// const maxLengthToPrint = 10000// debug(getByTestId('messages'), maxLengthToPrint, {highlight: false})

rerender#

类型：function

如果你想测试一个已渲染的组件在其 props 更新时的一些场景，你可以使用 rerender 来现实。

import { render } from '@modern-js/runtime/testing';
const { rerender } = render(<NumberDisplay number={1} />)
// re-render the same component with different propsrerender(<NumberDisplay number={2} />)

unmount#

类型：function

unmount 会卸载掉已渲染的组件。如果你想测试组件卸载后的情况（如，绑定的事件是否在 unmount 阶段被卸载掉），那么这个 API 是很帮助的。

unmount 相当于执行了 ReactDOM.unmountComponentAtNode

import { render } from '@modern-js/runtime/testing';
const { container, unmount } = render(<Login />)unmount()// your component has been unmounted and now: container.innerHTML === ''

asFragment#

类型：function

asFragment 函数返回当前渲染的组件的 DocumentFragment 对象。可用于测试 react 事件触发后 DOM 结构的响应。

import React, { useState } from 'react'import { render, fireEvent } from '@testing-library/react'
const TestComponent = () => {  const [count, setCounter] = useState(0)
  return (    <button onClick={() => setCounter((count) => count + 1)}>      Click to increase: {count}    </button>  )}
const { getByText, asFragment } = render(<TestComponent />)const firstRender = asFragment()
fireEvent.click(getByText(/Click to increase/))
// 这里会产生一份第一次渲染，和触发了点击事件后第二次渲染后的 DOM 结构之间差异的快照。// See https://github.com/jest-community/snapshot-diffexpect(firstRender).toMatchDiffSnapshot(asFragment())
