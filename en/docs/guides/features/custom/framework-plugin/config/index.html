插件与配置

Modern.js 是配置驱动的，所以在 Modern.js 提供了统一的方式进行配置维护、校验和获取。

维护#

这里主要在 modern.config.ts 中进行维护，Modern.js 中提供了 defineConfig 方法用于开发者设置用户配置。

modern.config.ts
import { defineConfig } from '@modern-js/app-tools'
export default defineConfig({  /** add some config */});
note

需要注意的是，不同的工程方案导出 defineConfig 方法是不一样的，目前有以下工程方案：

  • MWA：@modern-js/app-tools
  • 模块工程方案：@modern-js/module-tools
  • Monorepo 工程方案：@modern-js/monorepo-tools

校验#

可以通过【基础插件 Hook】中提到的 validateSchema 方法对支持的配置进行扩展，大概方式是以下面这样创建一个插件并添加：

config/plugin/fooPlugin.ts
import { createPlugin } from '@modern-js/core'
export default createPlugin(() => {  return {    validateSchema() {      return {        target: 'foo',        schema: {          type: 'string',        },      },    },  },}}

类型扩展：

config/plugin/fooPlugin.ts
declare module '@modern-js/core' {  export interface UserConfig {    foo: string;  }}

这样就添加了一个 string 类型的 'foo' 字段。

在 modern.config.ts 中这样配置：

modern.config.ts
import { defineConfig } from '@modern-js/app-tools'
export default defineConfig({  foo: 'demo message'});

获取#

在解析处理之后，通过 【上下文共享机制】来做到在任何插件中都可以获取配置，Modern.js 中提供了三个 API 用来获取应用上下文和配置：

  • useConfigContext: 应用原始配置
  • useAppContext: 应用上下文
  • useResolvedConfigContext: 配置

在插件中使用：

import { createPlugin, useAppContext, useResolvedConfigContext } from '@modern-js/core'
export default createPlugin(() => {  return {    prepare() {      const sourceConfig = useConfigContext()      const config = useResolvedConfigContext()      const context = useAppContext()
      /** do something */    },  }}}
