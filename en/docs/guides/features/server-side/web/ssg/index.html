一体化 SSG

SSG（static site generation）是一种基于数据与模板，在构建时渲染完整静态网页的工具，Modern.js 框架同样提供了开箱即用的使用方式。

note

SSG 仅作用在生产环境中，开发环境表现与 SSR 相同。

我们首先需要为应用添加 SSG 的依赖：

pnpm add @modern-js/plugin-ssg

前一小节介绍过，Modern.js 中支持两种路由方案约定式路由与自控式路由。

Modern.js 框架提供了 output.ssg 配置来开启 SSG 功能，在不同路由规则的 MWA 项目中需要使用不同配置方式。

在约定式路由中使用#

约定式路由是由 Modern.js 根据入口文件结构生成的路由，因此框架能够收集所有路由信息。

假设现在项目拥有一个主入口，并且有如下目录结构：

├── src/│   ├── pages/│   │   ├── home.tsx│   │   ├── info.tsx│   │   └── index.tsx

约定式路由，我们在配置中添加 ssg: true 开启应用的 SSG 功能：

modern.config.js
module.exports = {  output: {    ssg: true,  },}

约定式路由都会默认开启 SSG，现在简单的在上述文件中添加组件代码：

src/pages/index.tsx
export default () => {  return  <div>index page</div>}

SSG 和 SSR 一样，也是在 Node.js 环境中完成页面渲染，因此我们可以在开发阶段开启 SSR，保证不会出现开发阶段能正常运行，生产环境渲染失败的问题：

module.exports = {  server: {    ssr: process.env.NODE_ENV === 'development' ? true : false,  },  output: {    ssg: true,  }}

现在，在本地执行 dev 命令，等待构建完成后，查看 dist/ 目录下，此时应该只有一个入口对应的一涨 HTML 页面：

约定式路由的客户端路由都被渲染成了一张 HTML。

并且，页面的内容已经被渲染出来，可以看到 HTML 中存在 "info page" 字样。

执行 pnpm run start 启动项目后，访问页面，在 Preview 视图中可以看到页面已经完成渲染。

在自控式路由中使用#

自控式路由是开发者通过组件代码自定义的结构，同时也是运行时的结果。因此，在构建阶段，Modern.js 无法准确收集用户所有的路由信息。

例如下方代码，希望能够将 /foo 路由开启 SSG，但该 React Router 路由为运行时状态，Modern.js 无法准确感知。

import { Switch, Route } from '@modern-js/runtime/router';
export default () => (  <Switch>    <Route path="/" exact={true}>      <div>Home</div>    </Route>      <div>Foo</div>    <Route path="/foo" exact={true}>    </Route>  </Switch>);

此时用户需要告知 Modern.js 框架哪些路由需要开启 SSG，同样利用 output.ssg 配置。该参数允许配置一个钩子函数，调用 Modern.js 提供的 API 开启指定路由的 SSG 功能。

note

约定式路由，则每个可被识别的路由都会触发一次 ssg 钩子。通过 API 添加的页面会以当前触发钩子的入口的为基础来渲染，因此需要在对应入口下添加页面。

下方配置首先判断当前触发钩子的入口，接着调用 createPage 将 /foo 路由添加为 SSG 页面，并将渲染结果输出到 ./foo.html 中。

import { defineConfig } from '@modern-js/app-tools'
export default defineConfig({  output: {    ssg(context) {      const { route, createPage } = context;      if (route.path === '/') {        createPage({          url: '/foo'          output: './foo.html',        });      }    },  },})

执行 pnpm run build 与 pnpm run start 后刷新页面后，访问 http://localhost:8080/foo，在 Preview 视图中可以看到页面已经完成渲染。

查看产物文件，可以看到 dist/ 目录中，在默认的 main 入口产物目录下，新增一张 foo.html 的页面：

note

约定式路由默认开启 SSG 的行为，相关内容可以查看 API 文档。

获取静态数据#

在 SSR 中，组件可以通过 useLoader 同构的获取数据。在 SSG 中，Modern.js 也提供了相同的能力。

开发者可以通过 useLoader 第二个参数中可选的添加 { static: true } 请求数据，让数据获取可选的执行在 SSG、SSR 或 CSR 阶段。

note
  • Modern.js 目前还不支持 SSG 与 SSR 混合渲染，敬请期待。
  • 在开发阶段，不论 useLoader 是否配置 { static: true }，函数都会在 SSR 时获取数据。

修改上述 App.ts 的代码为：

import { Switch, Route } from '@modern-js/runtime/router';import { useLoader } from '@modern-js/runtime';
export default () => {  const { data } = useLoader(    async () => ({      message: Math.random(),    }),    {      static: true,      params: 'foo',    },  );
  return (    <Switch>      <Route path="/" exact={true}>        <div>Home</div>      </Route>      <Route path="/foo" exact={true}>        <div>Foo, {data?.message}</div>      </Route>    </Switch>  );};

执行 pnpm run dev，重复刷新页面，可以看到 /foo 页面的渲染结果不断发生变化，说明数据是在请求时获取的。

重新执行 pnpm run build 后，执行 pnpm run start，重复刷新页面，发现页面渲染结果始终保持同样的内容，说明数据在请求时不会再次获取。
