定义路由

Modern.js 在运行时是一个 Universal App（大统一应用）——— 既有客户端部分，又有服务端部分。但是开发者对服务端部分是无感知的，不需要专门去为 Server 部分编写代码，只需要专注产品本身即可。

开发者无需像传统的以服务端为中心的框架那样，定义每条路由。Modern.js 会为每一个入口都会生成一个 HTML 与路由，既能实现单页 Web 应用（SPA），也能实现多页网站。

其中，把由 Modern.js 根据入口名与配置，为入口生成对应的路由，也就是决定 HTML 响应的路由，称为 服务端路由。

把开发者在入口内部，通过文件约定定义的约定式路由或代码中定义的自控式路由称为 客户端路由。

服务端路由#

服务端路由是由 Modern.js 根据入口以及配置生成的路由。入口可以遵循多种文件约定，例如 App.j(t)sx 或 pages/ 等，但入口内部的文件约定并不影响服务端路由的生成，这里以前者来举例。

src/ 下如果有 App.j(t)sx，就认为这个应用是单入口的，src/ 下的其他文件和目录，都会被视为入口内部的文件。

例如以下结构，是生成默认入口名为 main 的单入口应用

.├── src/│   ├── .eslintrc.json│   ├── App.css│   └── App.jsx

上述结构将会生成一条路径为 / 的服务端路由。

src/ 下如果没有 App.j(t)sx，那么 src/ 下的任何一级目录里，如果有 App.j(t)sx，这个目录就会被视为一个应用入口。

例如以下结构，是入口名为 contacts 与 landing-page 的多入口应用：

.├── src/│   ├── contacts/│   │   ├── App.css│   │   └── App.jsx│   ├── landing-page/│   │   ├── App.css│   │   └── App.jsx│   └── .eslintrc.json

上述结构将会生成两条服务端路由，分别为 /contacts 和 /landing-page。

note

服务端路由中还会带有是否为 SSR 等额外信息。

开发者可以通过在 modern.config.js 中配置 server.routes 来修改默认生成的服务端路由。

客户端路由#

客户端路由是开发者通过入口内的文件约定来定义，或在代码中直接组织的路由。

其中，通过代码组织的称为【 自控式路由 】，通常使用 react-router 来实现，例如：

App.tsx
import { Switch, Route } from '@modern-js/runtime/router'
export default () => (  <Switch>    <Route path="/" exact={true}>      <div>Hello Modern.js</div>    </Route>    <Route path="/info" exact={true}>      <div>User info</div>    </Route>    <Route path="*">      <div>404</div>    </Route>  </Switch>)

上述代码将会生成两条确定的客户端路由 / 和 /info，其余所有未匹配到的路由都会返回 404。

通过文件约定定义的称为【 约定式路由 】，即使用 pages/ 组织的入口，例如：

.├── src/│   ├── pages/│   │   ├── _app.tsx│   │   ├── docs.tsx│   │   └── index.tsx│   └── .eslintrc.json

上述目录结构将会生成一个【 约定式路由 】的入口，包含 / 与 /docs 两条客户端路由。

又例如以下多入口的情况：

├── src/│   ├── contacts/│   │   └── pages/│   │       ├── _app.tsx│   │       ├── docs.tsx│   │       └── home.tsx│   ├── landing-page/│   │   └── pages/│   │       ├── _app.tsx│   │       ├── docs.tsx│   │       └── index.tsx│   ├── .eslintrc.json

该目录结构将会生成两个【 约定式路由 】的入口，每个入口对应两条客户端路由：

  • contacts 入口：/contacts/home、/cotacts/docs。

  • landing-page 入口：/landing-page、/landing-page/docs。

note

约定式路由底层同样基于 React Router。
